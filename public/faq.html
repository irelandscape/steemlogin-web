Frequently Asked Questions

What information about me does SteemLogin store?

SteemLogin stores your Steem username and private posting key.
Nothing else.

What if I enter the wrong key by accident?

SteemLogin automatically verifies that users enter an existing Steem username along with the matching private posting key.
The user won't be able to store any other key or invalid details.

Where does SteemLogin store my Steem details?

Steem usernames and private posting keys are stored within Cloud Firestore, a database service provided by Firebase and part of the Google Cloud Platform.

Cloud Firestores automatically encrypts and decrypts data as it is stored and retrieved from the database.

Database rules prevent a user (authenticated or not) to access the Steem username and posting key of another user.

What happens if I change my posting key?

SteemLogin verifies the validity of the Steem username and posting key upon each user authentication.

If the Steem details verification fails, the user is prompted to enter a new valid Steem username and posting key.

What information does SteemLogin provide to the Steem application?

SteemLogin supplies the Steem username and private posting key of the authenticated user to the Steem application along with the authentication provider identity (e.g. 'google.com') and the authentication tokens provided by this authentication provider.

The authorization scope requested by SteemLogin to the authentication provider is minimalistic at this stage. That is, the authentication tokens returned to the Steem application provide access to very few services theoritically offered by the authentication provider.

Future versions of SteemLogin may expand on this and allow the Steem application to request wider authorization scopes from the authentication providers. This could be used in the future to provide better integration between the Steem applications and Google, Facebook, etc.

Do you provide an SDK?

No, SteemLogin doesn't provide any software development kit.

Instead your application should be able to:

  • Redirect the user to the SteemLogin authentication URL.
  • Service an authentication success/failure URL over HTTPs
  • Initiate an HTTPs GET request to retrieve the user details

This is explained in details on our Developers page.

Is SteemLogin opensource?

Yes, the source code can be downloaded and examined from our GitHub repository.
